The importance of immutable backups against ransomware

Ransomware has become one of the biggest threats for companies of all sizes.

When an attack enters the network, it can encrypt servers, databases, documents, local backups, and critical systems. In many cases, attackers also try to delete or modify backups before encrypting the main information.

That is why having a backup is no longer enough. Companies need protected, external, and, whenever possible, immutable backups.

What is an immutable backup

An immutable backup is a backup file that cannot be modified, overwritten, or deleted during a defined retention period.

This means that even if an attacker gains administrator credentials or access to part of the infrastructure, they cannot delete or alter those protected copies.

Immutability adds a crucial line of defense against ransomware attacks.

Why traditional backups are not enough

For years, many companies have relied on local backups on USB drives, NAS devices, or internal servers.

The problem is that if those copies are permanently connected to the network, they can also be compromised during an attack.

Common risks include:

  • Backups accessible from the same Active Directory domain.
  • NAS storage with compromised credentials.
  • Permanently mapped network drives.
  • Backups without version control.
  • Unmonitored backup jobs.
  • Lack of restore testing.
  • Backup copies stored only in the physical office.

A backup that cannot be restored is not real protection.

The 3-2-1 Rule

A basic backup strategy should follow the 3-2-1 rule:

  • 3 copies of your data.
  • 2 different media types or locations.
  • 1 copy stored off-site.

Today, it is recommended to add another layer of protection to this rule: immutability.

This is why modern strategies refer to the 3-2-1-1-0 rule:

  • 3 copies.
  • 2 different media.
  • 1 off-site copy.
  • 1 immutable or air-gapped copy.
  • 0 errors verified through regular restore tests.

Benefits of immutable backups

Immutable backups help:

  • Protect against malicious deletion by hackers or disgruntled employees.
  • Recover clean data after a ransomware incident.
  • Reduce downtime and the impact of an attack.
  • Maintain historical file versions.
  • Improve business continuity plans.
  • Avoid depending on paying a ransom to recover operations.
  • Provide solid recovery guarantees.

Local, cloud, and protected repositories

A good backup architecture combines different layers:

  • Local backups for fast recovery.
  • Cloud backups for off-site protection.
  • Immutable repositories for critical recovery scenarios.
  • Continuous monitoring of backup tasks.
  • Automated alerts for failures.
  • Periodic restore testing.

The exact combination depends on the size of the business, data volume, system criticality, and budget.

What needs to be protected

You shouldn't just copy files. In a business, the following assets are also critical:

  • Full server images.
  • Virtual machines.
  • SQL and ERP databases.
  • Microsoft 365 emails.
  • SharePoint sites and OneDrive files.
  • Business management software.
  • ERP data (Sage, SAP, A3).
  • Firewall configurations.
  • Administrative records.

Each environment requires a tailored backup policy.

Microsoft 365 also needs a backup

Many companies believe that using Microsoft 365 means their data is automatically backed up. Microsoft guarantees platform availability, but it does not replace a dedicated corporate backup strategy.

It is highly recommended to run external backups for:

  • Exchange Online mailboxes.
  • SharePoint documents.
  • OneDrive folders.
  • Microsoft Teams chats and shared files.

This helps retrieve deleted files, previous versions, or data affected by human error or security incidents.

Restore testing

A backup is only as good as its restore capability.

We recommend reviewing regularly:

  • That backup jobs run successfully.
  • That no errors or warnings are logged.
  • That backups are stored in the correct locations.
  • That individual files can be restored.
  • That full servers can be booted from backup images.
  • That recovery time objectives (RTO) are acceptable.

Conclusion

Ransomware has rewritten the rules of backup.

It is no longer enough to just have a backup copy somewhere. Companies need verified, offsite, and tamper-proof backups.

An immutable backup strategy can be the difference between recovering your operations or facing a critical business shutdown.
